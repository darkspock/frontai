# Listas de verificación operacionales

## Nuevo feature

### Antes de la implementación
- [ ] Spec del feature creada en `working/[feature]/spec.md`.
- [ ] Agente validador ejecutado — spec `Status: validated`.
- [ ] Lista de tareas generada en `working/[feature]/tasks.md`.

### Durante la implementación
- [ ] Cada tarea ejecutada en orden, una por una.
- [ ] Cada tarea marcada `[x]` antes de empezar la siguiente.
- [ ] Tareas bloqueadas anotadas con razón.

### Después de la implementación
- [ ] Todas las tareas marcadas `[x]` en `tasks.md`.
- [ ] Spec `Status` actualizado a `implemented`.
- [ ] Flujo de UX definido y funcionando.
- [ ] Breadcrumb implementado en pantallas internas.
- [ ] Título de página y breadcrumb alineados.
- [ ] Estados de UI cubiertos: loading / empty / error / success.
- [ ] `npm run lint` pasado.
- [ ] `npm run typecheck` pasado.
- [ ] `npm run build` pasado.
- [ ] Documentación sincronizada.

## Refactor

- [ ] Comportamiento funcional preservado.
- [ ] Navegación y rutas intactas.
- [ ] Breadcrumb y navegación hacia atrás intactos.
- [ ] Cobertura de tests suficiente.
- [ ] Complejidad reducida o igual.
- [ ] Riesgos documentados.

## Corrección de bug

- [ ] Causa raíz identificada.
- [ ] Caso reproducible documentado.
- [ ] Corrección validada con test.
- [ ] Estado de UI y mensaje de error validados.
- [ ] Nota de regresión añadida.
