# IA Docs

Documentacion operativa para asistentes de IA.

## Estructura sugerida
- `00-contexto-del-proyecto.md`: contexto tecnico de frontend y limites actuales.
- `01-reglas-del-agente.md`: reglas para implementar frontend con Codex/Claude Code.
- `02-flujo-de-ejecucion.md`: flujo paso a paso para tareas frontend.
- `03-estandares-de-codigo.md`: convenciones frontend y calidad minima.
- `04-checklists.md`: listas de control por tipo de cambio frontend.
- `05-prompts-base.md`: prompts reutilizables.
- `06-agente-validador.md`: agente que valida especificaciones en `working/` antes de implementar.
- `frontend/`: documentacion tecnica fuerte del frontend (arquitectura, ADR, gobernanza e integracion API).

## Regla de uso
- Este espacio se escribe para IA.
- Todo requisito tecnico operativo debe existir aqui antes de pedir implementacion.
- Esta carpeta es la fuente principal de documentacion tecnica del proyecto.
