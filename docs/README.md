# Docs

Documentacion orientada a personas para contexto de negocio y producto.

## Estructura sugerida
- `00-vision-y-objetivo.md`: problema, propuesta de valor y objetivos del frontend.
- `01-alcance-y-requisitos.md`: alcance, perfiles de usuario y requisitos funcionales/no funcionales.
- `04-roadmap.md`: plan por fases, entregables y riesgos.

## Regla de uso
- Este espacio se escribe para humanos.
- Mantener contenido ligero, orientado a contexto y decisiones de negocio.
- Evitar especificaciones tecnicas detalladas en estos archivos.
- La documentacion tecnica fuerte vive en `ia_docs/frontend`.
